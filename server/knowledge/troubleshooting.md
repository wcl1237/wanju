# 故障排查指南

## 常见错误及解决方案

### 错误代码 400：请求参数错误
**原因**：提交的请求参数格式不正确或缺少必填字段。
**解决方案**：
1. 检查请求的 JSON 格式是否正确
2. 确认所有必填字段已提供
3. 检查字段类型是否匹配（如数字字段不要传入字符串）
4. 参考 API 文档确认参数要求

### 错误代码 401：认证失败
**原因**：访问令牌无效、过期或缺失。
**解决方案**：
1. 检查 Authorization 头是否正确设置
2. 确认令牌是否已过期，如已过期请重新获取
3. 检查 API Key 是否被禁用
4. 确认使用的是正确环境的密钥（测试/生产）

### 错误代码 403：权限不足
**原因**：当前用户没有执行该操作的权限。
**解决方案**：
1. 确认当前用户的角色和权限
2. 联系项目管理员调整权限设置
3. 检查是否在正确的项目/空间下操作

### 错误代码 429：请求频率超限
**原因**：在短时间内发送了过多的请求。
**解决方案**：
1. 降低请求频率
2. 实现请求重试机制（使用指数退避策略）
3. 考虑升级到更高级别的账户以获得更高限额
4. 使用批量操作接口减少请求次数

### 错误代码 500：服务器内部错误
**原因**：服务器遇到了意外错误。
**解决方案**：
1. 稍后重试请求
2. 检查服务状态页面了解是否有已知故障
3. 如问题持续存在，请联系技术支持并提供请求 ID

## 性能优化

### 页面加载缓慢
1. 清除浏览器缓存和 Cookie
2. 检查网络连接是否稳定
3. 尝试使用其他浏览器
4. 禁用不必要的浏览器扩展

### API 响应时间长
1. 减少单次请求的数据量（使用分页）
2. 使用字段筛选，只请求需要的数据
3. 考虑使用缓存减少重复请求
4. 检查是否存在 N+1 查询问题

### 同步数据延迟
1. 检查 Webhook 配置是否正确
2. 确认网络连通性
3. 查看同步日志排查具体错误
4. 如使用定时同步，确认 Cron 表达式正确
